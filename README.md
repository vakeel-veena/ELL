# Engineering Leadership Lab (ELL) – Program Playbook

This repository is the **single source of truth for designing, operating, and analyzing the Engineering Leadership Lab (ELL) pilot program**, supported by structured use of Large Language Models (LLMs).

It captures program context, operational playbooks, prompts, decisions, and metrics used to run the program.

If something is **not documented here, it is not considered part of the program record.**

---

# Program Overview

The **Engineering Leadership Lab (ELL)** is a leadership development pilot designed to accelerate the growth of high-potential individual contributors toward enterprise-level leadership behaviors.

The program focuses on developing:

- Confidence in speaking up
- Strategic thinking
- Visibility of impact
- Assertive communication
- Ownership and accountability

Participants are high-performing engineers who deliver technically but need stronger enterprise leadership presence.

# Repository Structure

## `context/`

Stable program context used by both humans and LLMs.

Examples:
- program purpose
- cohort definition
- leadership behaviors being developed
- program structure and timeline

These files provide **baseline understanding of the program.**

---

## `playbook/`

Operational rules for running the program.

Defines:
- participant health logic
- intervention framework
- roles of pod managers and mentors
- program governance

These documents explain **how the program operates.**

---

## `metrics/`

Program measurement and evaluation.

Contains:
- participant engagement signals
- program progress indicators
- cohort health tracking
- final outcome measures

This section helps determine whether the program is **actually working.**

---

## `prompts/`

Standard prompts used with LLMs.

Examples include prompts for:

- participant health analysis
- cohort pattern detection
- leadership signal extraction
- intervention suggestions

Prompts are designed to ensure **consistent and repeatable LLM usage.**

---

## `changes/`

Decision and change log for the program.

Tracks:
- program design updates
- metric adjustments
- prompt revisions
- operational changes

This preserves the **history of the pilot program.**

---

## `MasterContext.md`

The master context file used by LLMs.

- mission
- leadership behaviors targeted

# Operating Principle

**If it is not recorded in this repository, it does not exist.**

All of the following must be documented here:

- program rules
- structural changes
- prompts used for analysis
- program decisions
- measurement definitions

This ensures transparency, repeatability, and institutional learning.

---

# Program Operating Rhythm

The ELL program follows a structured cycle:

1. Pod managers observe participant engagement and leadership behaviors.
2. Selected participants are reviewed through structured discussion.
3. Interventions are assigned to accelerate leadership development.
4. Signals are analyzed using LLM-supported workflows.
5. Outcomes and decisions are recorded in this repository.

---

# Why This Repository Exists

Leadership programs often fail because:

- development signals are anecdotal
- interventions are inconsistent
- program outcomes are poorly documented

This repository creates a **structured system for leadership development**, combining human judgment with LLM-assisted analysis.
